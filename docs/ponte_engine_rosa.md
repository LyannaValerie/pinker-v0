# Ponte Engine ↔ Rosa

- **Classe:** Ponte
- **Papel:** híbrido
- **Status:** ativo

## 1. Objetivo

Conectar explicitamente:

- o **estado real implementado** (Engine);
- a **direção identitária/lexical/visionária** (Rosa).

## 2. Regras de conversa entre hemisférios

1. Nenhuma visão aspiracional pode ser apresentada como recurso pronto.
2. Nenhuma entrega técnica deve ignorar critérios de identidade lexical da Pinker.
3. Conceitos devem ter **um texto canônico** e referências cruzadas (sem duplicação difusa).
4. Rodadas documentais devem preservar separação entre navegação, estado, crônica e handoff.

## 3. Decisão prática por tipo de tema

- Tema de **estado, teste, fase, handoff, pipeline** → Engine.
- Tema de **vocabulário, voz, identidade, sintaxe futura, soberania estética** → Rosa.
- Tema que exige ambos (ex.: novas keywords, framing de intrínsecas, fronteira de roadmap vs visão) → Ponte.

## 4. Gate de consistência antes de encerrar rodada

Checklist mínimo:

- O texto diferencia claramente presente vs futuro?
- Há links cruzados entre Engine e Rosa?
- `history.md`, `agent_state.md` e `handoff_codex.md` foram atualizados quando cabível?
- `atlas.md` continua apontando para os canônicos corretos?
