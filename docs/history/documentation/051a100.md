### Doc-46 - formalização da trilha bare-metal e bootstrap do Bloco 20
- Rodada exclusivamente documental, sem mudança funcional em linguagem, compilador, runtime, `src/`, `tests` ou `examples`.
- A lacuna entre o backend nativo Linux atual e uma cadeia freestanding adulta foi promovida de inferência dispersa a trilha estrutural explícita do Bloco 20.
- Criado `docs/roadmap/bare_metal_bootstrap.md` com quatro frentes de convergência: BM-A (toolchain freestanding), BM-B (bootstrap/runtime autônomo), BM-C (boot/fronteira de hardware) e BM-D (produto de build, QEMU e gate de qualidade).
- O padrão anti-mínimo de `docs/expandir.md` foi adotado explicitamente: uma fase pode ter escopo delimitado, mas só fecha ao entregar uma fatia vertical utilizável com superfície, semântica, backend/runtime, diagnósticos, testes positivos e negativos, exemplo, documentação e evidência executável.
- Stubs, placeholders, geração isolada de `.o`, impressão serial avulsa ou boot único não são critérios suficientes de conclusão.
- `docs/roadmap/blocos/bloco_20.md` preservou o detalhe histórico do Eixo B e passou a posicionar a trilha depois da Faixa 3, em convergência com a Faixa 7, sem renumerar os 52 itens existentes.
- Os marcos de SO foram refinados: Marco SO 0 para boot freestanding verificável; Marco SO 1 para núcleo de baixo nível operacional; Marco SO 2 para kernel operacional; Marco SO 3 para subsistema de dispositivos.
- O backend atual permanece descrito honestamente como ELF Linux x86-64 System V ligado ao `pinker_rt`; nenhuma capacidade bare-metal foi declarada implementada.
- `docs/future.md` recebeu apenas o inventário das lacunas freestanding, sem assumir a função de roadmap.
- `docs/handoff_codex.md`, `docs/atlas.md`, `README.md` e `docs/roadmap/indice.md` foram sincronizados com a nova referência estrutural.
- Documentos atualizados: `README.md`, `docs/atlas.md`, `docs/roadmap.md`, `docs/roadmap/indice.md`, `docs/roadmap/blocos/bloco_20.md`, `docs/roadmap/bare_metal_bootstrap.md`, `docs/future.md`, `docs/handoff_codex.md`, `docs/history/documentation/indice.md`, `docs/history/documentation/051a100.md`.
